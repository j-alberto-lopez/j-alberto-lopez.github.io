Proyecto destacado

Servetracker

API REST orientada a hostelería y entornos TPV para centralizar productos, stock y tickets de venta. El proyecto pone el foco en la lógica de negocio: cálculo de ventas semanales, beneficio por producto, consumo de stock y datos útiles para analizar rentabilidad.

Java 17 Spring Boot REST JPA / Hibernate MySQL
Ver repositorio Volver a proyectos

Arquitectura

  • API independiente: backend Spring Boot pensado para ser consumido desde un frontend Angular.
  • Persistencia: Spring Data JPA con Hibernate y MySQL local para entidades de producto, ticket, pedido, receta, proveedor y stock.
  • Controladores: estructura por recurso con base REST, incluyendo gestión de productos y preparación para tickets y stock.
  • DTOs: respuestas orientadas a listado con métricas agregadas y detalle de proveedores con stock disponible.

Qué implementa

  • CRUD de productos mediante /api/v1/productos.
  • Consulta de productos con stock actual disponible.
  • Cálculo de ventas semanales comparadas con la semana anterior.
  • Cálculo de precio de compra medio ponderado y beneficio estimado por producto.
  • Base de entidades y relaciones para tickets, pedidos, proveedores, recetas, consumos y alertas.

Notas técnicas

  • El backend permite CORS desde http://localhost:4200, coherente con el frontend Angular previsto.
  • La lógica de productos combina repositorios de pedidos y tickets para calcular stock, ventas y beneficio.
