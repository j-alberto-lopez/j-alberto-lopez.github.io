Recetario

Aplicación web orientada a la gestión de recetas con arquitectura separada entre backend REST y cliente servidor-renderizado. El proyecto resuelve autenticación, administración básica de usuarios y operaciones CRUD sobre recetas y niveles de dificultad.

Java 17 Spring Boot Thymeleaf MySQL JWT
Ver API Ver cliente web Volver a proyectos

Arquitectura

  • recetas-api: API REST con Spring Boot, Spring Data JPA y persistencia en MySQL.
  • recetas-cli: cliente web con Thymeleaf que consume la API mediante RestTemplate.
  • Autenticación: login con generación y validación de JWT entre cliente y backend.
  • Modelo: entidades principales para recetas, dificultades, usuarios y roles.

Qué implementa

  • CRUD de recetas con asociación a dificultad.
  • Gestión básica de dificultades desde interfaz web.
  • Login de usuarios con token y sesión.
  • Panel de administración con listado, alta, borrado y modificación de usuarios.
  • Endpoints REST separados por recurso para recetas, dificultades, usuarios y roles.

Validación realizada

  • He revisado la separación cliente/API y el flujo principal de login, recetas, dificultades y menú admin.
  • La API compila y pasa mvnw.cmd test en este entorno.
  • El cliente Thymeleaf compila y pasa mvnw.cmd test en este entorno.
  • La configuración actual espera MySQL local en localhost:3306/recetas con usuario root.

Notas técnicas

  • La API publica por defecto en el puerto 9090.
  • El cliente consume la API localmente, por lo que no hay demo pública desplegada todavía.
  • El código fuente debe revisarse desde su repositorio propio, no empaquetado dentro del portfolio.
  • La solución está orientada a prácticas reales de backend Java y consumo de servicios REST desde MVC clásico.
